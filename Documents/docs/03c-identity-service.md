# Identity Service Low-Level Deep Dive

The Identity Service (`identity-service`) manages user registration, authentication (via JWT), profiles, and roles (`CLIENT` and `SELLER`). It saves data in MongoDB and publishes registration events to Kafka.

---

## 1. Application Configuration (`application.yml`)
Path: [application.yml](file:///Users/aung.min/Desktop/github/buy-01-dev/identity-service/src/main/resources/application.yml)

### Configurations:
*   **`server.port: 8081`**: Runs on port `8081`.
*   **`spring.data.mongodb.uri`**: Connects to Mongo database `buy01` on host `mongodb:27017`.
*   **`spring.data.mongodb.auto-index-creation: true`**: Creates unique index key mappings automatically.
*   **`spring.kafka.bootstrap-servers: kafka:9092`**: Connects to the Kafka broker.
*   **`jwt.secret`**: Cryptographic secret key for signing generated JWTs.

### `UserService.java`
Path: [UserService.java](file:///Users/aung.min/Desktop/github/buy-01-dev/identity-service/src/main/java/com/example/identity/service/UserService.java)

#### Properties:
*   `passwordEncoder`: An instance of `BCryptPasswordEncoder`.
*   `kafkaTemplate`: Directs event messages to Kafka topics.

#### Methods:
*   `public User register(String name, String email, String password, Role role)`:
    *   **Logic**:
        1.  Verifies if email is already taken. If so, throws `DuplicateEmailException`.
        2.  Encodes the password. Since the password arrives from the frontend already SHA-256 hashed, it encodes it as `BCrypt(SHA256(plaintext))`.
        3.  Sets model properties and calls `userRepository.save(user)`.
        4.  Publishes user ID event to the `"user-registered"` Kafka topic.
        5.  Returns the saved User instance.
*   `public Optional<User> authenticate(String email, String password)`:
    *   **Logic**: Looks up user by email. Compares the incoming password (SHA-256 hash) with the saved BCrypt hashed password using `passwordEncoder.matches()`. Returns `Optional.of(user)` if correct, otherwise `Optional.empty()`.
*   `public Optional<User> findByEmail(String email)`:
    *   **Logic**: Returns user mapped to matching email.
*   `public Optional<User> updateProfile(String email, String name)`:
    *   **Logic**: Finds user by email, changes name if provided, and saves.
*   `public Optional<User> updateProfileWithPassword(...)`:
    *   **Logic**: Finds user by email. Sequentially invokes `validateAndUpdatePassword`, `updateNameIfProvided`, and `validateAndUpdateEmail` helper methods, then saves.
*   `private void validateAndUpdatePassword(User user, String oldPassword, String newPassword)`:
    *   **Logic**: Validates that old password matches database record. If valid, hashes new password and assigns it to the user.
*   `private void validateAndUpdateEmail(User user, String currentEmail, String newEmail)`:
    *   **Logic**: If changing email, checks if new email is already registered to another account. If not, updates email field.

### `JwtService.java`
Path: [JwtService.java](file:///Users/aung.min/Desktop/github/buy-01-dev/identity-service/src/main/java/com/example/identity/service/JwtService.java)

#### Properties:
*   `key`: Holds the HMAC-SHA signing key.

#### Methods:
*   `public JwtService(@Value("${jwt.secret}") String secret)`:
    *   **Logic**: Initializes key using `SecretKeySpec` passing the secret bytes and `HS256` signature algorithm identifier.
*   `public String generateToken(User user)`:
    *   **Logic**: Maps claims (`userId`, `role`). Sets subject to `user.email`, issued timestamp to now, expiration to 24 hours from now (`86,400,000` ms). Signs with `key` and compacts to token string.

## 4. Controllers & Security

### `AuthController.java`
Path: [AuthController.java](file:///Users/aung.min/Desktop/github/buy-01-dev/identity-service/src/main/java/com/example/identity/controller/AuthController.java)

#### Methods:
*   `public ResponseEntity<Object> register(@RequestBody Map<String, String> req)`:
    *   **Endpoint**: `POST /register`
    *   **Logic**: Extracts `name`, `email`, `password`, and `role`. Calls `userService.register()`. Returns registered details with a success message. Catches exceptions and returns HTTP 400 or HTTP 500 error maps.
*   `public ResponseEntity<Object> login(@RequestBody Map<String, String> req)`:
    *   **Endpoint**: `POST /login`
    *   **Logic**: Extracts credentials. Calls `userService.authenticate()`. If success, calls `jwtService.generateToken()` and returns token alongside user metadata. Otherwise returns HTTP 401 Unauthorized.
*   `public ResponseEntity<Object> getProfile(Authentication auth)`:
    *   **Endpoint**: `GET /me` (Requires authentication)
    *   **Logic**: Extracts username/email from Spring security principal. Looks up user details and returns them.
*   `public ResponseEntity<Object> updateProfile(...)`:
    *   **Endpoint**: `PUT /me` (Requires authentication)
    *   **Logic**: Extract changes from body. Calls `userService.updateProfileWithPassword()` to apply changes.

### `HeaderAuthenticationFilter.java`
Path: [HeaderAuthenticationFilter.java](file:///Users/aung.min/Desktop/github/buy-01-dev/identity-service/src/main/java/com/example/identity/filter/HeaderAuthenticationFilter.java)
A servlet filter that reads headers forwarded by the API Gateway (`X-User-Id`, `X-User-Email`, `X-User-Role`) and builds the Spring Security authentication object.

#### Methods:
*   `protected void doFilterInternal(...)`:
    *   **Logic**: Extracts the headers. If `X-User-Id` is present:
        1.  Wraps roles inside a `SimpleGrantedAuthority` (prefixed with `ROLE_`).
        2.  Creates a `UsernamePasswordAuthenticationToken` using email as principal and authorities list.
        3.  Sets authentication context into `SecurityContextHolder`.
        4.  Calls `filterChain.doFilter` to continue servlet execution.

## 5. Test Summary

### Core Test Files:
1.  **`AuthControllerTest.java`**: Verifies login and registration paths. Asserts correct HTTP status codes on validation failures, duplicate email registrations, and password mismatches.
2.  **`JwtServiceTest.java`**: Verifies JWT generation logic. Parses tokens using verification keys to assert that claims are encoded correctly and expiration matches 24 hours.
3.  **`HeaderAuthenticationFilterTest.java`**: Simulates incoming gateway requests containing `X-User-Id` headers and asserts that security context is populated with correct authorities.
4.  **`UserTest.java`**: Confirms MongoDB model properties and default constructor initializations.

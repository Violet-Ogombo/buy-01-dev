# Product Service Low-Level Deep Dive

The Product Service (`product-service`) manages the catalog metadata, wishlist items, search indexing/filtering, and aggregates seller profiles. It communicates with other services synchronously (validating images uploaded to the `media-service` via REST) and asynchronously (consuming image upload events via Kafka).

## 3. Services

### `ProductService.java`
Path: [ProductService.java](file:///Users/aung.min/Desktop/github/buy-01-dev/product-service/src/main/java/com/example/product/service/ProductService.java)

#### Methods:
*   `public void reduceStock(List<ReduceStockItem> items)`:
    *   **Logic**: Loops through list items. Fetches product from database. If stock `quantity < requested`, throws `IllegalArgumentException`. Subtracts requested value from `quantity`, increments `salesCount`, and adds `price * quantity` to `revenue`. Saves model.
*   `public void revertStock(List<RevertStockItem> items)`:
    *   **Logic**: Invoked when an order gets cancelled. Re-adds items back to product inventory (`quantity += request`). Decrements `salesCount` (clamped to 0). Subtracts subtotal from `revenue` (clamped to 0). Saves model.
*   `public List<Product> getAllProducts(int page, int size)`:
    *   **Logic**: Requests page sorted descending by `createdAt`. Returns list.
*   `public Optional<Product> getProductById(String id)`:
    *   **Logic**: Fetches product.
*   `public Product createProduct(ProductCreateRequest request, String userId)`:
    *   **Logic**: Maps DTO to model, sets defaults (`salesCount=0`, `revenue=0`, new list if `imageUrls` is null), logs "CREATE" to `AuditService`, sends message to Kafka topic `"product-created"`.
*   `public Product updateProduct(String id, ProductCreateRequest request, String userId)`:
    *   **Logic**: Finds product. Verifies owner (`product.userId == userId`). If unauthorized, throws `AccessDeniedException`. Updates properties, logs "UPDATE" audit trail, sends message to Kafka topic `"product-updated"`.
*   `public void deleteProduct(String id, String userId)`:
    *   **Logic**: Deletes product from database if user is the product owner. Logs "DELETE" audit and sends Kafka message `"product-deleted"`.
*   `public Product addImages(String productId, List<String> mediaIds, String userId)`:
    *   **Logic**: Adds image URLs to a product by verifying the images exist and belong to the correct user:
        1.  Loops over media IDs.
        2.  Executes REST call to `media-service` endpoint `/images/{mediaId}/meta` using `RestTemplate`.
        3.  Asserts that returned `RemoteMedia` is not null and `media.userId` matches the current `userId`.
        4.  Generates image relative path (`/api/media/images/{mediaId}`) and appends it to product `imageUrls` list if not already present. Saves changes.

### `SearchService.java`
Path: [SearchService.java](file:///Users/aung.min/Desktop/github/buy-01-dev/product-service/src/main/java/com/example/product/service/SearchService.java)

#### Methods:
*   `public List<ProductSearchDTO> searchByKeyword(String keyword)`:
    *   **Logic**: Returns all products if query empty. Otherwise, filters product list where lowercase title/description matches keyword. Sorts descending by `salesCount`.
*   `public List<ProductSearchDTO> filterProducts(BigDecimal minPrice, BigDecimal maxPrice)`:
    *   **Logic**: Filters products within price range. Sorts ascending by price.
*   `public List<ProductSearchDTO> filterAndSearch(...)`:
    *   **Logic**: Filters by keyword, category, and price range. Sorts by popularity.
*   `public List<ProductSearchDTO> getFilteredAndSorted(...)`:
    *   **Logic**: Filters by category, price, then sorts using `sortProducts` helper method (supporting: `price_asc`, `price_desc`, `name`, `newest`, `popularity`).

### `SellerProfileService.java`
Path: [SellerProfileService.java](file:///Users/aung.min/Desktop/github/buy-01-dev/product-service/src/main/java/com/example/product/service/SellerProfileService.java)

#### Methods:
*   `public SellerProfileDTO getSellerProfile(String sellerId)`:
    *   **Logic**: Fetches products where `userId == sellerId`. Calculates aggregate metrics:
        *   `totalProducts` = list size.
        *   `totalSales` = sum of `salesCount` of all products.
        *   `totalRevenue` = sum of `revenue` of all products.
        *   `bestSellingProducts` = sorts products descending by `salesCount` and limits to top 5.

## 4. Controllers

*   **`ProductController.java`**: Matches `/` pathways. Exposes GET all (paginated), GET by ID, POST to add (Requires SELLER role), PUT to update (owner check), DELETE to remove (owner check), POST to `/images` to bind uploads, and internal endpoints `/reduce-stock` & `/revert-stock` invoked by `order-service` during transactions.
*   **`SearchController.java`**: Exposes `/api/v1/products/search`, `/filter`, `/search-and-filter`, and `/filter-and-sort` GET endpoints. Permits all users.
*   **`SellerProfileController.java`**: Exposes GET `/api/v1/sellers/{sellerId}/profile` returning sales statistics.
*   **`WishlistController.java`**: Exposes GET wishlist items, POST to add, DELETE to remove, and GET `/count` endpoints. Requires authentication.
